Portfólio

Modelo de criação de calçados utilizando redes GANs

O projeto foi desenvolvido para a empresa Beira Rio Calçados com o objetivo de auxiliar no processo criativo para lançamento de novas peças. O modelo foi criado utilizando a linguagem de programação Python e redes GANs (Stable Diffusion).

A técnica de Stable Diffusion foi escolhida para treinar a GAN utilizada no modelo. Esta técnica é baseada em amostragem em cadeia de Markov e foi escolhida por sua capacidade de gerar imagens de alta qualidade a partir de imagens de entrada.

O modelo desenvolvido tem como entrada duas imagens de calçados e como saída uma nova imagem de calçado gerada pela rede GAN. Para a criação do modelo, foi utilizada uma GAN já pré-treinada, que foi adaptada para o uso específico na criação de calçados.

Para o deploy do modelo, foi utilizado o API Flask e AWS Elastic Container Service, que permitem a criação e o gerenciamento de serviços em nuvem de maneira rápida e eficiente.

O modelo de criação de calçados utilizando redes GANs foi um sucesso para a Beira Rio Calçados, pois permitiu a criação de novas peças com mais rapidez e eficiência. Além disso, o modelo também ajudou na redução de custos com prototipagem de calçados, já que a geração de imagens em 3D a partir do modelo foi mais rápida e precisa do que a criação física de protótipos.

Ferramentas/Tecnologias utilizadas: Linguagem de Programação (Python), GANs (Stable Diffusion), Deploy (API Flask, AWS Elastic Container Service).

Modelo de recomendação de calçados por similaridade

O projeto foi desenvolvido utilizando técnicas avançadas de deep learning e visão computacional. A linguagem de programação utilizada foi Python, que é amplamente utilizada em projetos de inteligência artificial e machine learning.

O primeiro passo do projeto foi tratar e padronizar as imagens do banco de dados da Beira Rio Calçados, que continha mais de 30 mil imagens. Isso foi feito utilizando técnicas de pré-processamento de imagens, como redimensionamento, normalização e correção de brilho e contraste.

Em seguida, foi utilizado o modelo de classificação Zero Shot Images de deep learning para classificar as imagens. Esse modelo foi treinado com uma grande quantidade de imagens e foi capaz de classificar as imagens em várias categorias, incluindo cor, estilo, forma e material.

Para buscar imagens semelhantes, foi utilizado o TensorFlow Similarity, uma biblioteca de visão computacional que é capaz de encontrar imagens similares a partir de uma imagem de referência. Esse modelo utiliza técnicas de processamento de imagens, como extração de características, para identificar as imagens mais semelhantes.

Por fim, o modelo foi implementado em uma API Flask e implantado em um serviço de contêineres AWS Elastic Container Service, permitindo que a equipe da Beira Rio Calçados pudesse acessá-lo facilmente e fazer buscas de imagens semelhantes em seu banco de dados. Com isso, a empresa foi capaz de encontrar referências mais rapidamente, auxiliando no processo criativo e desenvolvimento de novas peças de calçados.

Ferramentas/Tecnologias utilizadas: Linguagem de Programação (Python), Deep learning (Classificação - Zero Shot Images), Visão Computacional (Tensorflow Similarity), Deploy (API Flask, AWS Elastic Container Service).

Probabilidade de churn com análise de sobrevivência

O projeto teve como objetivo identificar se diferentes taxas de engajamento dos clientes estavam relacionadas a uma menor probabilidade de churn. Para isso, foram utilizadas ferramentas e tecnologias como Linguagem de Programação Python, Análise de Sobrevivência com a biblioteca Lifelines, Tratamento de Dados com Pandas e Visualização de Dados com Seaborn, Matplotlib e Storytelling.

O conceito de engajamento dos usuários adotados na análise foi definido como aqueles que retornaram ao cliente em um intervalo menor ou igual a 3 meses após o seu primeiro uso. Foram analisadas diferentes taxas de engajamento dos clientes e a curva de sobrevivência foi utilizada para verificar a relação entre engajamento e probabilidade de churn.

A curva de sobrevivência é uma técnica estatística utilizada para analisar a probabilidade de um evento ocorrer em um determinado período de tempo. No caso deste projeto, o evento analisado foi o churn, que é quando um cliente deixa de utilizar um serviço ou produto. A curva de sobrevivência foi utilizada para calcular a probabilidade de um cliente permanecer utilizando o produto ao longo do tempo, de acordo com sua taxa de engajamento.

Através da análise da curva de sobrevivência, foi possível verificar que os clientes com uma taxa de engajamento entre 69% a 100% apresentavam uma melhor curva de sobrevivência em relação aos demais, indicando uma menor probabilidade de churn. Esse resultado foi verificado através de testes estatísticos como log-rank, cox coefficient p-value e survival in time t, que apontaram para uma diferença estatisticamente significativa entre as curvas de sobrevivência dos clientes com diferentes taxas de engajamento.

Em resumo, o projeto "Probabilidade de churn com análise de sobrevivência" utilizou a curva de sobrevivência para analisar a relação entre a taxa de engajamento dos clientes e a probabilidade de churn. Essa análise foi realizada utilizando ferramentas e tecnologias como Python, Lifelines, Pandas e Seaborn, permitindo a identificação de clientes com menor probabilidade de churn e auxiliando na tomada de decisões estratégicas da empresa.

Ferramentas/Tecnologias utilizadas: Linguagem de Programação (Python), Análise de Sobrevivência (Lifelines), Tratamento de Dados (Pandas), Visualização de Dados (Seaborn, Matplotlib, Storytelling).

Segmentação das personas de usuários de produtos

O projeto foi desenvolvido para resolver a hipótese de que é possível segmentar os tipos de comportamentos dos usuários para analisar o engajamento com o produto de maneira mais assertiva. O projeto foi dividido em quatro fases.

Na primeira fase, houve uma validação dos eventos dos usuários, em conjunto com a área de negócio, para definir quais eventos são importantes para a segmentação dos comportamentos. Na segunda fase, foi feita uma exploração dos eventos selecionados para avaliar, através dos dados, a utilização na segmentação dos comportamentos.

A terceira fase envolveu a aplicação de técnicas de clusterização para realizar a segmentação dos comportamentos dos usuários. Para isso, foi utilizada a ferramenta Scikit Learn, que é uma biblioteca de aprendizado de máquina em Python. Foi também aplicada a técnica de redução de dimensionalidade UMAP, que ajudou a visualizar os dados em duas dimensões.

Por fim, na quarta fase, foram caracterizados os comportamentos obtidos através do modelo de clusterização. Como resultado, foram encontradas quatro personas distintas: Cluster 1 com comportamento de uso relacionado ao envio de mensagens ativas; Cluster 2 com comportamento de uso de monitoramento; Cluster 3 com comportamento de uso relacionado ao envio de mensagens ativas pela aba de Growth; e Cluster 4 com comportamento de uso diversificado, porém com foco em desenvolvimento.

Durante todo o projeto, foram utilizadas ferramentas e tecnologias como Linguagem de Programação Python, Tratamento de Dados com Pandas, Visualização de Dados com Seaborn, Matplotlib e Storytelling, além das já mencionadas Scikit Learn e UMAP. A partir dos resultados obtidos, a empresa pôde utilizar as personas encontradas para entender melhor o engajamento dos usuários com o produto e direcionar melhorias e inovações.

Ferramentas/Tecnologias utilizadas: Linguagem de Programação (Python), Clusterização (Scikit Learn), Redução de dimensionalidade (UMAP), Tratamento de Dados (Pandas), Visualização de Dados (Seaborn, Matplotlib, Storytelling).

Score para avaliação do atendimento de atendentes

O projeto teve como objetivo criar um score para avaliar o desempenho dos atendentes em relação às métricas de atendimento. Para isso, foram utilizadas técnicas de redução de dimensionalidade, mais especificamente a Análise de Componentes Principais (PCA).

Inicialmente, foram definidas as métricas que comporiam o score: mediana do tempo de atendimento por ticket no trimestre, mediana do tempo de espera por ticket no trimestre e mediana do volume diário de tickets no trimestre. Essas métricas foram escolhidas por refletirem o desempenho dos atendentes em relação à eficiência no atendimento e à capacidade de lidar com volume de demanda.

Em seguida, foi realizada a análise de componentes principais (PCA), que tem como objetivo reduzir a dimensionalidade dos dados, mantendo a maior parte da variação dos dados originais. A técnica de PCA permite transformar as métricas em um conjunto menor de variáveis que melhor representam o desempenho dos atendentes.

Após a aplicação da técnica de PCA, foram gerados os scores para cada atendente. O score final foi calculado como uma combinação linear das variáveis principais obtidas a partir da PCA. Esses scores foram então utilizados para identificar os atendentes com baixa performance e indicar possíveis intervenções para melhorar seu desempenho, como treinamentos e realocações.

Por fim, a visualização dos scores dos atendentes em relação às métricas de atendimento permitiu aos gestores identificar padrões de desempenho e identificar oportunidades de melhoria no atendimento. A utilização do score criado com a técnica de PCA permitiu uma avaliação mais objetiva e consistente do desempenho dos atendentes, contribuindo para o aumento da eficiência do atendimento e, consequentemente, da receita dos clientes.

Ferramentas/Tecnologias utilizadas: Linguagem de Programação (Python), Redução de dimensionalidade (PCA), Tratamento de Dados (Pandas), Visualização de Dados (Seaborn, Matplotlib, Storytelling).

Modelo de classificação de risco de cancelamento cirúrgico

O projeto teve como objetivo criar uma nova funcionalidade para um produto que consistia em prever quais cirurgias poderiam ter um potencial de cancelamento a partir de um conjunto de informações. Para isso, utilizamos a linguagem de programação Python e a biblioteca Pycaret para desenvolver o modelo de classificação.

Inicialmente, fizemos o tratamento dos dados, utilizando a biblioteca Pandas para limpar e transformar os dados em um formato adequado para o modelo. Em seguida, utilizamos o Pycaret para realizar o treinamento do modelo de gradient boosting machine, que mostrou um F1-score de 0.71.

O modelo de gradient boosting machine é um algoritmo de aprendizado de máquina que utiliza uma combinação de árvores de decisão para realizar a classificação. Esse algoritmo é conhecido por sua capacidade de lidar com dados complexos e não lineares, o que o torna ideal para nosso problema. Apesar de termos feito vários treinamentos utilizando Pycaret.

Ao final do projeto, conseguimos criar um modelo de classificação eficiente que foi capaz de prever com precisão quais cirurgias tinham um potencial de cancelamento. Com isso, nossos clientes puderam utilizar essa nova funcionalidade para evitar cancelamentos desnecessários e aumentar a eficiência em suas operações.

Ferramentas/Tecnologias utilizadas: Linguagem de Programação (Python), Classificação (Pycaret), Tratamento de Dados (Pandas).

Modelo de classificação de satisfação de pacientes utilizando NLP

O projeto foi desenvolvido com o objetivo de identificar a satisfação dos pacientes e capturar elementos importantes de suas queixas ou elogios. O modelo foi integrado em uma solução ponta a ponta que trabalha com a satisfação dos pacientes e era requisitado via chatbot, analisando em tempo real as mensagens dos pacientes e direcionando para as áreas responsáveis.

As técnicas/estratégias utilizadas incluíram remoção de stopwords, tokenização, normalização de texto, remoção de pontuações, evitando flexões e derivações nas palavras e TF-IDF. Para o treinamento do modelo, utilizou-se o algoritmo de classificação Logistic Regression, que mostrou-se eficaz na classificação da satisfação dos pacientes obtendo uma acurácia de 72%.

O modelo desenvolvido foi capaz de classificar com precisão a satisfação dos pacientes, indicando se a mensagem do paciente era positiva (promotor) ou negativa (detrator). Além disso, o modelo também foi capaz de identificar os principais termos utilizados pelos pacientes em suas mensagens, permitindo que a equipe responsável pudesse tomar ações imediatas para melhorar a experiência do paciente.

O modelo foi implantado com sucesso em produção e integrado na solução desenvolvida para trabalhar a satisfação dos pacientes, trazendo resultados positivos para a empresa. A implementação do modelo permitiu uma análise mais precisa e em tempo real das mensagens dos pacientes, permitindo a tomada de ações imediatas para melhorar a experiência do paciente.

Ferramentas/Tecnologias utilizadas: Linguagem de Programação (Python), NLP (Scikit Learn, nltk), Tratamento de Dados (Pandas), Visualização de Dados (Seaborn, Matplotlib, Wordcloud), Deploy (API Flask, GCP APP Engine, Heroku).
